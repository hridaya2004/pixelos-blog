---
title: Bypassing FRP Lock
description: In this guide, we will see how to bypass the Factory Reset Protection (FRP) lock on Android devices.
---

This guide walks you through wiping the FRP lock on your device using ADB.

<Callout type="warn" title="WARNING">
  This method only works on **userdebug** builds.
</Callout>

Before starting, ensure your device is in **recovery mode** and you have [platform tools](https://developer.android.com/tools/releases/platform-tools) installed.

---

## Step 1: Verify ADB connection and gain root access

Check that your device is detected, then restart ADB as root:

```sh
adb devices
adb root
```

Expected output:

```sh
List of devices attached
XYZ123456789    recovery

restarting adbd as root
```

---

## Step 2: Run the script

<Callout type="info">If your build already includes the script, run it directly:</Callout>

```sh
adb shell wipe-frp
```

<Callout type="info">If your build does not include the script, follow the steps below.</Callout>

1. [Download `wipe-frp.sh`](https://raw.githubusercontent.com/PixelOS-AOSP/vendor_aosp/refs/heads/fifteen/prebuilt/common/bin/wipe-frp.sh).

2. Push the script to your device:

```sh
adb push wipe-frp.sh /sdcard/
```

Expected output:

```sh
wipe-frp.sh: 1 file pushed, 0 skipped. 0.1 MB/s (1686 bytes in 0.015s)
```

3. Make the script executable and run it:

```sh
adb shell chmod +x /sdcard/wipe-frp.sh
adb shell /sdcard/wipe-frp.sh
```

Expected output:

```log
device:/ # ./sdcard/wipe-frp.sh
32+0 records in
32+0 records out
32 bytes (32 B) copied, 0.004 s, 7.8 K/s
1000+0 records in
1000+0 records out
1000 bytes (0.9 K) copied, 0.004 s, 244 K/s
32+0 records in
32+0 records out
32 bytes (32 B) copied, 0.001 s, 31 K/s
8+0 records in
8+0 records out
8 bytes (8 B) copied, 0.001 s, 7.8 K/s
32+0 records in
32+0 records out
32 bytes (32 B) copied, 0.004 s, 7.8 K/s
device:/ #
```

---

## Step 3: Reboot your device

```sh
adb reboot
```

Your device should now boot without the FRP lock.
